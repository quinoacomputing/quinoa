Reporting Zoltan Bugs

The Zoltan team uses Bugzilla to collect bug reports. See the instructions below for using Bugzilla (adapted with permission from the Trilinos project).

To get started with Bugzilla for the first time:

  • Go to the Sandia Software Bugzilla page.
  • Click on "Open a new Bugzilla account."
  • Enter your e-mail address and name.
  • Check your e-mail for your new password.
  • Sign in with your new password.
  • Configure your password and preferences how you want them. This can be done under "Change password or user preferences" on the main page, or under "Edit prefs" in the footer.
To submit a bug, fill out a Bugzilla bug report:
  • Go to the Sandia Software Bugzilla page.
  • Click "Enter a new bug report" on the main page, or "New" in the footer.
  • Choose "Zoltan" from the list of packages.
  • Read the "bug writing guidelines," check the list of "most frequently reported bugs," and "search" for the bug to see if the bug has already been submitted (in which case, it may be helpful to add a new comment to the existing report).
  • Select the version number, component, platform, operating system, priority, and severity of the bug.
  • Enter a concise, informative summary.
  • Describe, in as much detail as possible, the feature you are requesting or the bug/issue you are reporting.
  • Press the "Commit" button at the bottom of the page.
  • After submitting your bug report, you should receive email confirming the submission. All Zoltan developers will also receive the bug report.
To find a bug, run a Bugzilla query:
  • Go to the Sandia Software Bugzilla page.
  • Click "Search existing bug reports" on the main page, or "Search" in the footer.
  • Select appropriate entries from any relevant fields.
  • Click the "Search" button.
